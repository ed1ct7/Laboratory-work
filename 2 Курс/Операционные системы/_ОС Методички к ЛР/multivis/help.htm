Рабочее окно приложения.

В основном окне отображается процесс моделирования, осуществляется доступ к элементам меню.

1.     File -> New – создание нового набора заданий.

2.     File -> Open – открытие существующего набора.

3.     File -> Edit – редактирование уже открытого набора заданий (например, после выполнения моделирования).

Меню View служит для управления процессом моделирования, описание команд приводится в разделе «Моделирование».

 

Диалог «Ввод заданий».

Диалог предназначен для ввода и редактирования набора заданий и параметров моделируемой системы.

1.     Аппаратные средства – задание параметров моделируемой системы: оперативной памяти и количества внешних устройств.

2.     Задания – список заданий для моделируемой системы.

3.     Дисциплина обслуживания – задание дисциплины обслуживания (FIFO, LIFO, SJF, PRT).

4.     Старт – запуск процесса моделирования. По умолчанию, моделирование выполняется в режиме временной диаграммы.

5.     Выход – выход из режима редактирования набора заданий без запуска режима моделирования, с закрытием текущего файла.

6.     Добавить – добавление нового задания (см. описание диалога «Редактирование задания»).

7.     Удаление – удаление выделенного задания.

8.     Редактирования – редактирование выделенного задания.

9.     Сохранить – сохранение набора заданий в файле (файл с расширением dat).

10. Помощь – вызов этой справки.

 

Диалог «Редактирование задания».

Диалоговое окно предназначено для редактирования и ввода параметров уже существующих или новых заданий.

1.     Время поступления – время поступления задания в систему.

2.     Трудоемкость – количество процессорного времени, необходимого для выполнения задания.

3.     Оперативная память – количество оперативной памяти, необходимое для выполнения задания.

4.     Внешние устройства – количество внешних устройств, необходимых для ввода задания.

5.     Приоритет – приоритет задания (учитывается только при использовании дисциплины PRT).

 

Моделирование.

Процесс моделирования может отображаться в 2-х вариантах. Переключение между ними осуществляется через меню View.

1.     View -> Diagram – режим моделирования, используемый по умолчанию. Используется для наглядного представления результатов моделирования.

2.     View -> Process – режим, позволяющий проследить за всем процессом моделирования. В этом режиме схематично отображается ход выполнения различных заданий.

3.     View -> Replay – повтор моделирования.

4.     View -> Increase Speed (Decrease Speed) – увеличение (уменьшения) скорости отображения информации о моделировании (для режима Diagram).

5.     Statistics – отображение статистики моделирования (отображается время поступления, начала и окончания выполнения задания, а также средневзвешенное время).

Время ввода каждого задания определим по формуле:

tввода = q × N , где

q – время работы одного внешнего устройства (q=5 ед.времени), N – число внешних устройств.

Таким образом, если бы задания выполнялись по одному, то на каждое задание было бы затрачено время

T = tввода + tпроц. , где

T – время выполнения задания,

tпроц. – процессорное время, затраченное для выполнения задания.

В ЭВМ поступает поток заданий, поэтому при одновременном выполнении нескольких заданий процессорное время распределяется между ними равномерно. Т.о. реально затраченное время для выполнения задания ≥ необходимого процессорного времени.

Для простоты предположим, что все внешние устройства могут работать параллельно, не задерживая друг друга.

Каким образом можно проверить правильность построения времен­ной диаграммы?

Для этого необходимо определить по временной диаграмме, сколько времени работал процессор. Это временные интервалы с t=5 до t=65 и с t=75 до t=85. В интервале с t=65 до t=75 процессор простаивал. Таким образом, процессор работал в течение 70 ед. времени.

Далее необходимо просуммировать процессорное время всех выполненных заданий.

Tпроцессора = 30+10+10+20=70 ед.

Значения равны, следовательно, диаграмма построена верно.

Для оценки эффективности дисциплины обслуживания найдет средневзвешенное время обращения. Согласно формуле . Необходимо найти взвешенные времена выполнения каждого задания (данные берем из временной диаграммы и таблицы):

1.      W1 = (tЗ – tП) / T = (tЗ – tП) / (tввода + tпроц) = (tЗ – tП) / (q × N + tпроц) = (45 – 0) / (5 × 1 + 40) = 1.00

2.      W2 = (65 – 7) / (5 × 1 + 15) = 2.90

3.      W3 = (82 – 12) / (5 × 2 + 10) = 3.65

4.      W4 = (60 – 20) / (5 × 1 + 35) = 1.00

Т.о. Wср = (W1 + W2 + W3 + W4) / 4 = 2.14.
